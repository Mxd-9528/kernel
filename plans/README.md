# 前端架构演进方案

kernel 的终端显示经历了多次修补——`\x1b[1A` 清除序列、input() 回显、Rich 渲染——这些问题的根源是同一个：**把终端当作渲染面，而终端不是为交互式富文本显示设计的**。

以下三个方案是从"修补终端"到"正确抽象"的演进路径。方案 A 已排除，方案 B 为近期目标，方案 C 为远期规划。

## 方案概览

| 方案 | 思路 | 状态 |
|------|------|------|
| [A - TUI 框架](A-textual.md) | 用 Textual 替代 Rich，在终端内实现正确的交互布局 | ❌ 已排除 |
| [B - WebSocket + Web 前端](B-websocket.md) | observer 事件流通过 WebSocket 发送到浏览器前端 | ✅ 当前目标 |
| [C - IPC 进程分离](C-ipc.md) | agent 作为独立服务进程，前端可以是任意客户端 | 📋 远期规划 |
